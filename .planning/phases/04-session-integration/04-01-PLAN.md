    Replace old claude() function with new implementation:

    New logic:
    1. Parse arguments:
       - --help: Show usage
       - --version: Show version
       - --new: Force new project (skip selector)
       - Other args: Pass to claude CLI

    2. Detect context:
       - Is user already in a project session? Check if pwd contains ~/.claude/projects/
       - If yes: Launch claude with args (don't select)
       - If no: Continue to selection

    3. Project selection:
       - Call: selected=$( ~/.claude/scripts/project-select.sh quick )
       - If selected is empty: User cancelled, return
       - If selected is "NEW": Launch new project creation flow
       - If selected is a valid path: Use that path

    4. Switch to project:
       - cd "$selected" || return 1
       - Update registry: ~/.claude/scripts/registry-update.sh update_last_accessed "$project_id"
       - Update session count: ~/.claude/scripts/registry-update.sh increment_session_count "$project_id"
       - Show brief message: "📁 Resuming: {display_name}"

    5. Launch claude CLI:
       - command claude "$@"
       - This launches the real claude CLI with any arguments passed

    Error handling:
    - If project-select.sh not found: Show error, exit gracefully
    - If registry-update.sh not found: Show warning, continue anyway
    - If cd fails: Show error, return without launching claude
    - Invalid paths: Skip silently

    Function should be ~50-80 lines (clean and maintainable).
  </action>
