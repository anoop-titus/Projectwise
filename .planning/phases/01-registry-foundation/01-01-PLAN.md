  <name>Task 1: Create .registry.json schema and initialize</name>
  <files>~/.claude/projects/.registry.json</files>
  <action>
    Create `.registry.json` with schema containing:
    - version: 1
    - projects: array of project objects
    - categories: array of default categories

    Each project object has:
    - id: slug identifier
    - folder_name: actual encoded folder name
    - display_name: human-friendly name
    - description: brief project description
    - tags: string array (e.g., ["mcp", "python"])
    - category: one of predefined categories
    - status: "active", "paused", or "archived"
    - created: ISO timestamp
    - last_accessed: ISO timestamp
    - session_count: number
    - git_link: optional git repo URL

    Default categories: ["Research", "Medicine", "Leisure", "Productivity", "Finance", "Travel", "Business", "Stay"]

    Initialize with empty projects array and default categories. This is a template that will be populated by the init script.
  </action>
  <verify>cat ~/.claude/projects/.registry.json | jq . succeeds and validates JSON</verify>
  <done>Schema valid, file created with version 1 and default categories</done>
</task>

<task type="auto">
  <name>Task 2: Create registry-init.sh to scan and index projects</name>
  <files>~/.claude/scripts/registry-init.sh</files>
  <action>
    Create bash script that:
    1. Scans ~/.claude/projects/ for directories (depth 1 only, exclude dotfiles and symlinks initially)
    2. For each folder, generates sensible display name:
       - Strip leading/trailing dashes
       - Replace double dashes with spaces
       - Convert to Title Case
       - Examples: "-Users-titus--claude-python" → "Claude Python"
                   "claude_1769760221" → "Claude Code Session (Jan 30)"
                   "-Users-titus-Library-CloudStorage-MediShift-iOS" → "MediShift iOS"
    3. Extract metadata from existing PROJECT.json if present
    4. Calculate stats: file count (via find), size (via du -sh), modified date (via stat)
    5. Detect git repo if present (check for .git folder)
    6. Mark recently modified projects as status "active" (< 7 days)
    7. Build JSON array and merge with existing .registry.json (preserve user edits to existing entries)
    8. Write atomically with .tmp file pattern

    Make script safe: no destructive operations, read-only scanning, use jq for JSON manipulation.
  </action>
  <verify>
    Run: ~/.claude/scripts/registry-init.sh
    Output shows: "Scanned X projects" and "Updated .registry.json"
    cat ~/.claude/projects/.registry.json | jq '.projects | length' shows 11
  </verify>
  <done>Script executes without errors, all 11 projects indexed in .registry.json with generated names and metadata</done>
</task>
